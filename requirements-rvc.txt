# Optional dependencies for the AI voice changer (Effets → Voix IA tab).
# Install with: pip install -r requirements-rvc.txt
#
# These are NOT bundled in the Q-Pad installer because torch alone weighs
# ~1.5 GB and would 30× the installer size. Dev-mode users (running
# `python main.py` from a clone) install them once and the Voix IA tab
# becomes functional.
#
# Stack: vendored RVC synthesizer (rvc_lib/) + HuggingFace transformers
# HuBERT (ContentVec) + RMVPE / torchcrepe / pyworld / parselmouth for
# pitch detection. No fairseq, no omegaconf, no hydra — installs cleanly
# on plain Python 3.11 + Windows wheels, no MSVC Build Tools needed.

torch
torchaudio
# Required by torch.onnx.export when we bake the user's .pth into a
# frozen ONNX graph for the DirectML / ONNX Runtime backend. Separate
# package from the onnxruntime / onnxruntime-directml install below
# (which is the actual inference runtime).
onnx
# After torch.onnx.export, we convert the synth graph to FP16 internally
# (keep_io_types=True, so callers still see FP32 IO). RDNA3 / Ampere+ run
# the inner conv/matmul stack ~1.5-2× faster in FP16 — meaningful on the
# live mic path. Optional: if missing, the export falls back to FP32 with
# a terminal warning.
onnxconverter-common
# Simplify the graph after torch.onnx.export: fuses constants, eliminates
# redundant Cast/Transpose chains, propagates static shape inference.
# Often removes the very Cast patterns that break the FP16 conversion
# above, and gives a 20-30% steady-state speed-up on DML even without
# the FP16 win — DML pays per-op launch overhead, fewer ops = faster.
onnx-simplifier
# transformers 4.50+ pulls torch.distributed.tensor.device_mesh which only
# landed in torch 2.5. If you've installed torch-directml (AMD/Intel GPU,
# see below), pip downgrades torch to 2.4.x and the latest transformers
# blows up on import. Pin under 4.50 to keep both happy.
transformers<4.50
faiss-cpu
librosa
soundfile
numpy
scipy
requests

# Pitch extractors — pick rmvpe in the UI for best quality.
torchcrepe          # 'crepe' pitch method
pyworld             # 'harvest' / 'dio' pitch methods
praat-parselmouth   # 'pm' pitch method

# ─── Optional GPU acceleration ─────────────────────────────────────────
# CPU inference is fine for offline 'Convertir un son', but the live mic
# voice changer needs GPU to keep up. Pick ONE of the following based on
# your hardware AFTER the base install above:
#
#   - AMD GPU on Windows (e.g. RX 7900 XTX/GRE, RX 6000 series):
#       pip install torch-directml onnxruntime-directml
#     torch-directml lets the engine list adapters and select your GPU.
#     onnxruntime-directml is the actual inference backend — torch+DML
#     segfaults the RVC synth, ONNX Runtime + DML is rock-solid (same
#     path w-okada uses on AMD).
#
#   - Intel Arc / iGPU on Windows: same as AMD (DirectML).
#
#   - NVIDIA GPU on Windows: replace `torch` above with the CUDA build,
#       pip uninstall -y torch torchaudio
#       pip install torch torchaudio --index-url https://download.pytorch.org/whl/cu121
#     Auto-detected as CUDA, no extra package needed.
